# MDX Style Guide (Demo)

This file demonstrates the common style components supported by our GitHub-flavored Markdown renderer in the webapp. Use it as a reference when writing new sections.

---

## Text and emphasis

Regular paragraph with **bold**, *italic*, and ~~strikethrough~~, plus `inline code`.

Escaping special characters: `\*not bold\*`, `\_not italic\_`, `` \`literal backticks\` ``.

> Blockquote for short callouts or quotations.

**Tip:** Prefer short, scannable sentences.
**Note:** Keep code semantics intact.
**Attention:** Avoid breaking changes and unsafe patterns.

## Code blocks

JavaScript:
~~~js
export function sum(a, b) {
  return a + b;
}
console.log(sum(2, 3));
~~~

TypeScript / TSX:
~~~tsx
type ButtonProps = { label: string; onClick?: () => void };

export default function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>;
}
~~~

HTML:
~~~html
<section class="card">
  <h2>Title</h2>
  <p>Content</p>
</section>
~~~

CSS:
~~~css
.card {
  padding: 1rem;
  border: 1px solid #ddd;
  border-radius: 12px;
}
.card h2 { font-size: 1.25rem; }
~~~

Shell:
~~~bash
# Install dependencies
npm install

# Start dev server
npm run dev
~~~

JSON:
~~~json
{
  "name": "demo",
  "private": true,
  "scripts": { "dev": "vite" }
}
~~~
