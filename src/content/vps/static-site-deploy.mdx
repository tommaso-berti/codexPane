# Static Website Deployment (Variant)

Use this page only for static-site specifics. For the shared deploy procedure, follow the golden path:

- [`VPS Deploy Golden Path (Caddy + Versioned Releases)`](/Users/tberti/WebstormProjects/codexPane/src/content/vps/deploy-golden-path.mdx)

## When to use this variant

Choose this variant when your app is plain HTML/CSS/JS and does not require a Node build step.

## What changes compared to the golden path

- Build step in CI can be skipped.
- Artifact source is the repo content (or pre-built static bundle).
- Caddy root still points to `/srv/www/<www-domain>/current`.

## Recommended flow

1. Keep the same release layout (`releases/current`).
2. Sync static files to `/releases/vX.Y.Z`.
3. Switch `current` symlink.
4. Reload Caddy.

## Minimal CI example (static artifact)

~~~yaml
- name: Resolve version
  run: |
    V=$( [ -f VERSION ] && tr -d ' \n\r' < VERSION || echo "0.1.0" )
    V=${V#v}
    echo "RELEASE_DIR=v$V" >> $GITHUB_ENV

- name: Upload static files
  run: |
    RR="/srv/www/<www-domain>/releases/${RELEASE_DIR}"
    ssh -o IdentitiesOnly=yes -i ~/.ssh/deploy_key "${{ secrets.VPS_USER }}@${{ secrets.VPS_HOST }}" "mkdir -p \"$RR\""
    rsync -az --delete -e "ssh -o IdentitiesOnly=yes -i ~/.ssh/deploy_key" ./ "${{ secrets.VPS_USER }}@${{ secrets.VPS_HOST }}:${RR}/" \
      --exclude .git --exclude .github --exclude node_modules

- name: Activate release and reload caddy
  run: |
    RB="/srv/www/<www-domain>"
    ssh -o IdentitiesOnly=yes -i ~/.ssh/deploy_key "${{ secrets.VPS_USER }}@${{ secrets.VPS_HOST }}" \
      "ln -sfn \"$RB/releases/${RELEASE_DIR}\" \"$RB/current\" && sudo -n systemctl reload caddy"
~~~

## Unified script usage for static mode

~~~bash
sudo bash scripts/deploy-vps-unified.sh \
  --app-type static \
  --server caddy \
  --strategy ci-artifact \
  --app-name <app-name> \
  --domain <domain> \
  --www-domain <www-domain> \
  --deploy-user <deploy-user>
~~~

## Optional NGINX mode

NGINX remains supported by the unified script, but Caddy + versioned releases is the primary recommended flow.
