## Key directories
**/node_modules**
All dependencies and sub-dependencies specified in `package.json`. You donâ€™t edit this folder directly; itâ€™s in `.gitignore`.

**/public**
Static assets served directly without processing. Copied to the build output during production.
- `vite.svg` â€” Vite logo (replace with your favicon if you like).

**/src**
Your application code:
- `/assets` â€” images and other assets that Vite can process
- `App.jsx` â€” main React component
- `App.css` â€” styles for `App`
- `main.jsx` â€” app entry point (like `index.js`/`index.jsx` elsewhere)
- `index.css` â€” global styles

> **Note:** Throughout the Learn React materials we refer to `index.jsx` as the entry file. Viteâ€™s template uses `main.jsx` but it serves the same purpose. If you prefer to follow the lessons exactly, you can rename `main.jsx` to `index.jsx` and update the script reference in `index.html`.

## Key files
**index.html** (at project root)
The HTML entry for your app. Vite injects your JavaScript during build. Edit it to add meta tags, change the page title, or include additional scripts/styles.

**package.json**
Project settings:
- Dependencies
- Scripts for development, build, and preview
- Other metadata

Common scripts:
- `dev` â€” starts the dev server
- `build` â€” creates a production build
- `preview` â€” serves the production build locally

**vite.config.js**
Vite configuration (includes the React plugin by default). Extend it to add:
- Import aliases
- Custom build options
- Additional plugins
- Proxy settings for API requests

**eslint.config.js**
ESLint configuration (Vite template ships with a basic React-recommended setup).

**.gitignore**
Tells Git which paths to ignore (e.g., `node_modules`, build output, environment files).

> **Tip:** As your app grows, add a `/components` directory for reusable components and a `/views` directory for route-level pages.
