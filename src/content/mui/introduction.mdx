# Introduction to MUI
[
MUI](https://mui.com/) is a comprehensive React component library that helps you build accessible, responsive, and visually consistent UIs fast. It started with Material Design (Material UI) and has grown into a suite that includes:
- **Material UI**: Ready-made components following Google’s Material Design.
- **Joy UI**: Elegant, slightly opinionated components with a neutral design language.
- **Base UI**: Unstyled, accessible primitives for total visual control.
- **MUI System**: Theming, layout primitives, and the `sx` prop.
- **MUI X**: Advanced components like **Data Grid** and **Charts**.

Whether you need a quick dashboard or a finely tuned design system, MUI scales with your complexity and keeps your code ergonomic and type-safe.

---

## Why mui

- **Speed**: Hundreds of polished components, icons, and patterns out of the box.
- **Design systems**: Centralized theming with tokens (palette, typography, spacing, shape).
- **Styling flexibility**: Use `sx`, CSS variables, Emotion, styled API, or unstyled Base UI.
- **Accessibility**: ARIA-friendly defaults and strong keyboard support.
- **TypeScript-first**: Excellent typings and IntelliSense.

> Tip: You can mix and match libraries—e.g., use Base UI for primitives and Joy UI for higher-level elements.

## Install and first render

Install Material UI (or Joy UI/Base UI depending on your path). The examples below use Material UI.

~~~bash
# Core + icons (Material UI)
npm i @mui/material @emotion/react @emotion/styled @mui/icons-material
~~~

Wrap your app in a theme provider and start composing:

~~~jsx
import * as React from "react";
import { createRoot } from "react-dom/client";
import { ThemeProvider, createTheme, CssBaseline, Button, Container } from "@mui/material";
import SaveIcon from "@mui/icons-material/Save";

const theme = createTheme({
  palette: {
    mode: "light",
    primary: { main: "#1976d2" },
    secondary: { main: "#9c27b0" }
  },
  shape: { borderRadius: 10 }
});

function App() {
  return (
    <ThemeProvider theme={theme}>
      <CssBaseline />
      <Container sx={{ py: 4 }}>
        <Button variant="contained" startIcon={<SaveIcon />}>
          Save changes
        </Button>
      </Container>
    </ThemeProvider>
  );
}

createRoot(document.getElementById("root")).render(<App />);
~~~

> Note: MUI uses Emotion by default. You can switch to styled-components—see the official guide: https://mui.com/material-ui/guides/styled-engine/

## The sx prop and system utilities

`sx` is a powerful, theme-aware styling prop. It accepts objects, arrays, and functions—so tokens, breakpoints, and CSS variables are easy to use.

~~~jsx
import { Box, Stack, Typography } from "@mui/material";

export default function Hero() {
  return (
    <Box
      sx={{
        bgcolor: "primary.main",
        color: "primary.contrastText",
        py: { xs: 4, md: 8 },
        borderRadius: 2
      }}
    >
      <Stack spacing={1} sx={{ maxWidth: 720, mx: "auto", px: 2 }}>
        <Typography variant="h3">Build faster with MUI</Typography>
        <Typography variant="body1" sx={{ opacity: 0.9 }}>
          Theme-aware styles, accessible components, and developer-friendly APIs.
        </Typography>
      </Stack>
    </Box>
  );
}
~~~

> Tip: Use the theme object inside `sx` with a callback: `sx={(theme) => ({ boxShadow: theme.shadows[3] })}`.

## Theming essentials

Tokens (palette, typography, spacing, shape) keep your UI consistent. You can extend the theme with your own design tokens.

~~~ts
import { createTheme } from "@mui/material/styles";

declare module "@mui/material/styles" {
  interface Theme {
    brand: { radiusLg: number };
  }
  interface ThemeOptions {
    brand?: { radiusLg?: number };
  }
}

export const theme = createTheme({
  palette: {
    mode: "dark",
    primary: { main: "#7c4dff" },
    background: { default: "#0f0f12", paper: "#141419" }
  },
  typography: { fontFamily: '"Inter", system-ui, Arial, sans-serif' },
  spacing: 8,
  brand: { radiusLg: 16 }
});
~~~

Use custom tokens inside components via `sx` or `useTheme()`.

## Layout and responsiveness

Breakpoints (`xs`, `sm`, `md`, `lg`, `xl`) are baked in.

~~~jsx
import { Grid, Paper } from "@mui/material";

function Cards() {
  return (
    <Grid container spacing={2}>
      {Array.from({ length: 6 }).map((_, i) => (
        <Grid key={i} item xs={12} sm={6} md={4}>
          <Paper sx={{ p: 2, height: 160 }}>Card {i + 1}</Paper>
        </Grid>
      ))}
    </Grid>
  );
}
~~~

> Note: Prefer `Stack` for simple one-dimensional layouts; use `Grid` for two-dimensional arrangements.

## Data grid and charts

For advanced data-heavy UIs, install MUI X components.

~~~bash
npm i @mui/x-data-grid
~~~

~~~jsx
import * as React from "react";
import { DataGrid } from "@mui/x-data-grid";

const rows = [
  { id: 1, name: "Ada", role: "Engineer" },
  { id: 2, name: "Alan", role: "Researcher" }
];

const columns = [
  { field: "id", headerName: "ID", width: 90 },
  { field: "name", headerName: "Name", flex: 1 },
  { field: "role", headerName: "Role", flex: 1 }
];

export default function TeamTable() {
  return (
    <div style={{ height: 320 }}>
      <DataGrid rows={rows} columns={columns} pageSizeOptions={[5]} disableRowSelectionOnClick />
    </div>
  );
}
~~~

See docs: https://mui.com/x/react-data-grid/

## Accessibility and rtl

MUI aims for WCAG-friendly defaults and full keyboard support. Enable right-to-left with a theme direction switch and cache provider setup:
https://mui.com/material-ui/guides/right-to-left/

> Attention: Always test focus order, color contrast, and screen-reader labels (`aria-*`) in your app’s context.

## When to choose joy ui or base ui

- **Joy UI**: If you want a tasteful default aesthetic with easy CSS variables and color schemes: https://mui.com/joy-ui/getting-started/
- **Base UI**: If you need unstyled, accessible primitives that you’ll theme yourself: https://mui.com/base-ui/getting-started/
