# Js Queues

Queues in JavaScript

```
const LinkedList = require('./LinkedList');

class Queue {
  constructor(maxSize = Infinity) {
    this.queue = new LinkedList();
    this.maxSize = maxSize;
    this.size = 0;
  }

  isEmpty() {
    return this.size === 0;
  }

  hasRoom() {
    return this.size < this.maxSize;
  }

  enqueue(data) {
    this.queue.addToTail(data);
    this.size++;
    console.log(`Added ${data} to queue! Queue size is now ${this.size}.`);
  }

  dequeue() {
    if (!this.isEmpty()) {
      const data = this.queue.removeHead();
      this.size--;
      console.log(`Removed ${data} from queue! Queue size is now ${this.size}.`);
      return data;
    } else {
      throw new Error("Queue is empty!");
    }
  }
}

module.exports = Queue;

```

## General concept
A **Queue** is a data structure that follows the rule:
**FIFO - First In, First Out**
The first element added to the queue is the first element removed.
Example:

```
enqueue(A) enqueue(B) enqueue(C) Queue: A B C

dequeue() Queue: B C

```

Operations always occur at two ends:
* **tail** → where elements are added
* **head** → where elements are removed
In this implementation the queue internally uses a **LinkedList**.
The Queue keeps track of:
* queue → the LinkedList storing the elements
* size → the number of elements currently in the queue
* maxSize → maximum number of elements allowed

## constructor()

```
constructor(maxSize = Infinity) {
  this.queue = new LinkedList();
  this.maxSize = maxSize;
  this.size = 0;
}

```

Creates a new queue.
Initial state:
* queue → empty LinkedList
* size → 0
* maxSize → maximum capacity of the queue
If no size is provided:

```
maxSize = Infinity

```

Meaning the queue is **unbounded**.
Example:

```
const q = new Queue(3)

```

Maximum of **3 elements** allowed.

## isEmpty()

```
isEmpty() {
  return this.size === 0;
}

```

Checks whether the queue is empty.
Returns:

```
true  -> if size is 0
false -> if there are elements

```

Example:

```
Queue: []
isEmpty() → true

Queue: [A B]
isEmpty() → false

```

Key idea:
Used to prevent **removing from an empty queue**.

## hasRoom()

```
hasRoom() {
  return this.size < this.maxSize;
}

```

Checks if the queue has space for another element.
Returns:

```
true  -> if size < maxSize
false -> if queue is full

```

Example (maxSize = 3):

```
Queue: [A B]
size = 2
hasRoom() → true

Queue: [A B C]
size = 3
hasRoom() → false

```

Key idea:
Used to prevent **queue overflow**.

## enqueue(data)

```
enqueue(data) {
  this.queue.addToTail(data);
  this.size++;
  console.log(`Added ${data} to queue! Queue size is now ${this.size}.`);
}

```

Adds a new element to the queue.
Steps:
1. Insert the element at the **tail of the LinkedList**
```
this.queue.addToTail(data)

```

1. Increase the queue size

```
this.size++

```

1. Log a message confirming the insertion
Example:

```
enqueue("eggs")

```

Output:

```
Added eggs to queue! Queue size is now 1.

```

Example state:

```
Before: []
After : [eggs]

```

Key idea:
Queue insertion always happens at the **tail**.
Time complexity:

```
O(1)

```

because LinkedList supports constant-time tail insertion.

## dequeue()

```
dequeue() {
  if (!this.isEmpty()) {
    const data = this.queue.removeHead();
    this.size--;
    console.log(`Removed ${data} from queue! Queue size is now ${this.size}.`);
    return data;
  } else {
    throw new Error("Queue is empty!");
  }
}

```

Removes the element at the **front of the queue**.
Steps:
1. Check if the queue is empty

```
if (!this.isEmpty())

```

1. Remove the **head of the LinkedList**
```
const data = this.queue.removeHead()

```

1. Decrease the queue size

```
this.size--

```

1. Log the removal
Example:

```
Removed eggs from queue! Queue size is now 0.

```

1. Return the removed value

```
return data

```

Example state:

```
Before: [A B C]
dequeue() After : [B C]
Returned value: A

```

## Error case
If the queue is empty:

```
throw new Error("Queue is empty!")

```

Example:

```
Queue: []
dequeue()

```

Result:

```
Error: Queue is empty!

```

Key idea:
This prevents **queue underflow**.

## module.exports

```
module.exports = Queue;

```

Exports the Queue class so it can be used in other files.
Example:

```
const Queue = require('./Queue');

```

## Example usage

```
const groceries = new Queue();

groceries.enqueue("eggs");
groceries.enqueue("milk");
groceries.enqueue("bread");

groceries.dequeue();

```

Output:

```
Added eggs to queue! Queue size is now 1.
Added milk to queue! Queue size is now 2.
Added bread to queue! Queue size is now 3.
Removed eggs from queue! Queue size is now 2.

```

Final queue state:

```
[milk bread]

```

## Key ideas to remember
Queue properties:

```
queue   → LinkedList storing nodes
size    → number of elements
maxSize → capacity limit

```

Queue operations:

```
enqueue → add to tail
dequeue → remove from head

```

Queue rules:

```
FIFO
First In → First Out

```
