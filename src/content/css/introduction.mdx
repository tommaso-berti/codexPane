# Introduction to css

Cascading Style Sheets (CSS) describe how HTML is presented—layout, colors, spacing, typography, and motion. You’ll use CSS to turn raw markup into polished, responsive interfaces.

**Core ideas you’ll use constantly**

- **Cascade and specificity** decide which rule wins.
- **Box model**: every element is content + padding + border + margin.
- **Layout systems**: normal flow, Flexbox, and Grid.
- **Responsive design** with fluid units and media queries.
- **Selectors** (including pseudo-classes/elements) target exactly what you need.
- **Custom properties (variables)** make design tokens reusable.
- **Transitions & animations** add motion.

**A tiny tour**

~~~css
/* Selectors & specificity */
.card > h2:hover::after {           /* element + pseudo-element */
  content: " ↗";
  color: hsl(220 80% 56%);
}

/* Box model + a sane default */
*,
*::before,
*::after { box-sizing: border-box; }
.card {
  padding: 1rem;            /* inside */
  border: 1px solid #ddd;   /* edge  */
  margin-block: 1rem;       /* outside (logical) */
}

/* Flexbox: one-dimensional layout */
.toolbar {
  display: flex;
  gap: .5rem;
  justify-content: space-between;
  align-items: center;
}

/* Grid: two-dimensional layout */
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
}

/* Custom properties (design tokens) */
:root {
  --brand: hsl(270 90% 60%);
  --radius: 12px;
}
.button {
  background: var(--brand);
  border-radius: var(--radius);
}

/* Responsive with media queries & fluid units */
@media (width <= 640px) {
  .gallery { grid-template-columns: 1fr 1fr; }
}
h1 { font-size: clamp(1.5rem, 2vw + 1rem, 3rem); }

/* Motion */
.button {
  transition: background-color .2s ease, transform .2s ease;
}
.button:active { transform: scale(.98); }
~~~

**Tip:** Prefer logical properties (`margin-block`, `padding-inline`) for better internationalization (RTL/LTR).

**Note:** Units cheat—**px** (absolute), **rem** (root-relative), **%/vw/vh** (fluid), **fr** (Grid fraction), **ch** (character width).

**Attention:** Overly specific selectors (e.g., long chains, `!important`) make code brittle. Favor small, composable class rules and variables.
