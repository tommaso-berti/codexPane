# Introduction

[SQL](https://www.postgresql.org/search/?q=SQL) , **S**tructured **Q**uery **L**anguage, is a programming language designed to manage data stored in [relational databases](https://www.postgresql.org/search/?q=relational+databases). SQL operates through simple, declarative statements. This keeps data accurate and secure, and helps maintain the integrity of [databases](https://www.postgresql.org/search/?q=databases) , regardless of size.
The SQL language is widely used today across web frameworks and database applications. Knowing SQL gives you the freedom to explore your data, and the power to make better decisions. By learning SQL, you will also learn concepts that apply to nearly every data storage system.
The statements covered in this course use SQLite Relational Database Management System [(RDBMS)](https://www.postgresql.org/search/?q=%28RDBMS%29). You can also access a glossary of all the [SQL commands](https://www.postgresql.org/search/?q=SQL+commands) taught in this course.

## Relational database
A *[relational database](https://www.postgresql.org/search/?q=relational+database)* is a [database](https://www.postgresql.org/search/?q=database) that organizes information into one or more tables. Here, the relational database contains one table.
A *table* is a collection of data organized into rows and columns. Tables are sometimes referred to as *relations*. Here the table is
     celebs
 .
A *column* is a set of data values of a particular type. Here,
     id
 ,
     name
 , and
     age
  are the columns.
A *row* is a single record in a table. The first row in the
     celebs
  table has:
* An id of 1
* A name of Justin Bieber
* An age of 29
All data stored in a relational database is of a certain [data type](https://www.postgresql.org/search/?q=data+type). Some of the most common data types are:
* INTEGER, a positive or negative whole number
* TEXT, a text string
* DATE, the date formatted as YYYY-MM-DD
* REAL, a decimal value
