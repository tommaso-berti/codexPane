# Keywords

## As (select)
[AS](https://www.postgresql.org/search/?q=AS) is a keyword in SQL that allows you to *rename* a column or table using an alias. The new name can be anything you want as long as you put it inside of single quotes. Here we renamed the name column as Titles.

```
SELECT name AS 'Titles'
FROM movies;

```

Some important things to note:
* Although it's not always necessary, it is considered best practice to surround your aliases with single quotes.
	* Note that this practice is specific to SQLite, the RDBMS used in this exercise. When you work with other RDBMSs, notably PostgreSQL, [no quotes or double quotes](https://www.postgresql.org/docs/current/queries-select-lists.html#QUERIES-COLUMN-LABELS) may be required in place of single quotes.
* When using AS, the columns are not being renamed in the table. The aliases only appear in the result.

## Distinct
[DISTINCT](https://www.postgresql.org/search/?q=DISTINCT) is used to return unique values in the output. It filters out all duplicate values in the specified column(s).

```
SELECT DISTINCT genre
FROM movies;

```

## Where
We can restrict our query results using the [WHERE](https://www.postgresql.org/search/?q=WHERE) clause in order to obtain only the information we want.

```
SELECT *
FROM movies
WHERE imdb_rating > 8;

```

1. The WHERE clause filters the result set to only include rows where the following *condition* is true.
2. imdb_rating &gt; 8 is the condition. Here, only rows with a value greater than 8 in the imdb_rating column will be returned.
The
     &gt;
  is an *operator*. Operators create a condition that can be evaluated as either *true* or *false*.
Comparison operators used with the
     WHERE
  clause are:
* = equal to
* != not equal to
* &gt; greater than
* &lt; less than
* &gt;= greater than or equal to
* &lt;= less than or equal to

## Like
Useful operator when you want to compare similar values.
How could we select all movies that start with ‘Se' and end with ‘en' and have exactly one character in the middle?

```
SELECT *
FROM movies
WHERE name LIKE 'Se_en';

//Seven and Se7en both match this pattern

```

* LIKE is a special operator used with the WHERE clause to search for a specific pattern in a column.
* name LIKE 'Se_en' is a condition evaluating the name column for a specific pattern.
* Se_en represents a pattern with a *wildcard* character.

### Wildcards (to be used with like)
_ -&gt; means you can substitute any individual character here without breaking the pattern. The names
     Seven
  and
     Se7en
  both match this pattern
% -&gt; matches zero or more missing characters in the pattern

```
SELECT *
FROM movies
WHERE name LIKE 'A%';

```

* A% matches all movies with names that begin with letter ‘A'
* %a matches all movies that end with ‘a
Here, any movie that *contains* the word ‘man' in its name will be returned in the result.

```
SELECT *
FROM movies
WHERE name LIKE '%man%';

```

LIKE is not case sensitive. ‘Batman' and ‘Man of Steel' will both appear in the result of the query above.

## NULL (is) Unknown values are indicated by NULL.
It is not possible to test for NULL values with comparison operators, such as = and !=
Instead, we will have to use these operators:
* [IS NULL](https://www.postgresql.org/search/?q=IS+NULL)
* [IS NOT NULL](https://www.postgresql.org/search/?q=IS+NOT+NULL)
```
SELECT name
FROM movies
WHERE imdb_rating IS NOT NULL;

```

Please, note that is commonly used only to match NULL values. If a match with other values types is required, use = instead of IS

## Between (where) The [BETWEEN](https://www.postgresql.org/search/?q=BETWEEN) operator is used in a [WHERE](https://www.postgresql.org/search/?q=WHERE) clause to filter the result set within a certain *range*. It accepts two values that are either numbers, text, or dates.
For example, this statement filters the result set to only include movies with years from 1990 up to, *and including* 1999.

```
SELECT *
FROM movies
WHERE year BETWEEN 1990 AND 1999;

```

In this statement, BETWEEN filters the result set to only include movies with names that begin with the letter ‘A' up to, *but not including* ones that begin with ‘J'.

```
SELECT *
FROM movies
WHERE name BETWEEN 'A' AND 'J';

```

However, if a movie has a name of simply ‘J', it would actually match. This is because BETWEEN goes *up to* the second value - up to ‘J'. So the movie named ‘J' would be included in the result set but not ‘Jaws'.

## AND (where) Sometimes we want to *combine multiple conditions* in a [WHERE](https://www.postgresql.org/search/?q=WHERE) clause to make the result set more specific and useful. One way of doing this is to use the [AND](https://www.postgresql.org/search/?q=AND) operator. Here, we use the AND operator to only return 90's romance movies.

```
SELECT *
FROM movies
WHERE year BETWEEN 1990 AND 1999
   AND genre = 'romance';

```

* year BETWEEN 1990 AND 1999 is the 1st condition.
* genre = 'romance' is the 2nd condition.
* AND combines the two conditions.
With AND, *both* conditions must be true for the row to be included in the result.

## OR (where) With OR, if *any* of the conditions are true, then the row is added to the result.

```
SELECT *
FROM movies
WHERE year > 2014
   OR genre = 'action';

```

* year &gt; 2014 is the 1st condition.
* genre = 'action' is the 2nd condition.
* OR combines the two conditions.

## Order By (only after where) We can *sort* the results using [ORDER BY](https://www.postgresql.org/search/?q=ORDER+BY), either alphabetically or numerically. Sorting the results often makes the data more useful and easier to analyze.

```
SELECT *
FROM movies
ORDER BY name;

```

* ORDER BY is a clause that indicates you want to sort the result set by a particular column.
* name is the specified column.
Sometimes we want to sort things in a decreasing order. For example, if we want to select all of the well-received movies, sorted from highest to lowest by their year:

```
SELECT *
FROM movies
WHERE imdb_rating > 8
ORDER BY year DESC;

```

* DESC is a keyword used in ORDER BY to sort the results in *descending order* (high to low or Z-A).
* ASC is a keyword used in ORDER BY to sort the results in *ascending* order (low to high or A-Z). It is the default one if not specified otherwise
The column that we ORDER BY doesn't even have to be one of the columns that we're displaying.
Note: ORDER BY always goes after WHERE (if WHERE is present).

## Limit
[LIMIT](https://www.postgresql.org/search/?q=LIMIT) is a clause that lets you specify the maximum number of rows the result set will have. This saves space on our screen and makes our queries run faster.

```
SELECT *
FROM movies
LIMIT 10;

```

LIMIT always goes at the very end of the query. Also, it is not supported in all SQL databases.

## Case (select) A [CASE](https://www.postgresql.org/search/?q=CASE) statement allows us to create different outputs (usually in the SELECT statement). It is SQL's way of handling [if-then](https://en.wikipedia.org/wiki/Conditional_(computer_programming)) logic.
Suppose we want to condense the ratings in
     movies
  to three levels:
* *If the rating is above 8, then it is Fantastic.**If the rating is above 6, then it is Poorly Received.**Else, Avoid at All Costs.*
```
SELECT name,
 CASE
  WHEN imdb_rating > 8 THEN 'Fantastic'
  WHEN imdb_rating > 6 THEN 'Poorly Received'
  ELSE 'Avoid at All Costs'
 END
FROM movies;

```

* Each WHEN tests a condition and the following THEN gives us the [string](https://www.postgresql.org/search/?q=string) if the condition is true.
* The ELSE gives us the string if *all* the above conditions are false.
* The CASE statement must end with END.
In the result, you have to scroll right because the column name is very long. To shorten it, we can rename the column to ‘Review' using AS:

```
SELECT name,
 CASE
  WHEN imdb_rating > 8 THEN 'Fantastic'
  WHEN imdb_rating > 6 THEN 'Poorly Received'
  ELSE 'Avoid at All Costs'
 END AS 'Review'
FROM movies;

```
